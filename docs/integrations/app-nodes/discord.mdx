# Discord

![Discord app nodes overview](./discord/node-overview.png)

Discord nodes let you send messages, manage channels and roles, handle members, and start scenarios from Discord events.

## Connection

Discord supports **OAuth** (Latenode shared bot) and **Access Token (Personal App)** (your bot).

![Connection type choice](./discord/connection-types.png)

### OAuth

You authorize Latenode's bot and add it to your server. No separate bot hosting on your side.

![Discord OAuth screen](./discord/oauth-sign-in.png)

![Pick a server](./discord/oauth-server-select.png)

<Steps>
  <Step>

### Sign in with Discord

Click **Sign in with Discord**, choose the server, and confirm permissions.

  </Step>
  <Step>

### Save the connection

Name the connection and save.

![Connection saved](./discord/connection-saved.png)

  </Step>
</Steps>

#### Server ID for channel lists

After OAuth, set **Server ID** so Latenode can load channels.

<Steps>
  <Step>

### Turn on Developer Mode

In Discord: **Settings → Advanced → Developer Mode**.

![Developer Mode](./discord/developer-mode-settings.png)

  </Step>
  <Step>

### Copy Server ID

Right-click the server in the sidebar → **Copy Server ID**.

![Copy Server ID](./discord/copy-server-id.png)

  </Step>
  <Step>

### Paste into the node

Switch **Guild ID** from **Select** to **Map**, paste the ID, then set the channel field (paste **Channel ID** if needed).

![Guild ID in Map mode](./discord/guild-id-map-mode.png)

![Channel ID](./discord/paste-channel-id.png)

  </Step>
</Steps>

### Access Token (Personal App)

Your bot from the [Discord Developer Portal](https://discord.com/developers/applications).

<Steps>
  <Step>

### Open your application

In the portal, open the app that owns the bot.

  </Step>
  <Step>

### Copy the bot token

**Bot** → copy **Token**.

  </Step>
  <Step>

### Paste into Latenode

Paste into **Bot Token**, name the connection, and save.

  </Step>
</Steps>

Add the bot to the target server with the right permissions before using it in a scenario.

## Actions

Most ID fields support **Select** (pick from a loaded list) or **Map** (paste the **numeric id** you copy in Discord, or reference data from another node). If lists stay empty, set **Guild ID** first and check **Connection** above (OAuth needs **Server ID**).

<Accordions type="multiple">
<Accordion title="Roles: Add Role">

Adds a role to a user on the server.

| Field | Description |
| --- | --- |
| Guild ID | Server where the bot is present. **Select** when the dropdown loads, or **Map** the numeric **server id** (same idea as **Server ID** in **Connection**). |
| User ID | Member to update. **Select** if the node lists users; otherwise **Map** with an ID from another node or from Discord (Developer Mode → right-click user → **Copy User ID**). |
| Available Role | Role to add. Usually **Select** from roles loaded for that server. |

</Accordion>
<Accordion title="Roles: Remove User Role">

**Removes a role** from a member on the server.

| Field | Description |
| --- | --- |
| Guild ID | Same as other actions: **Select** from the list or **Map** the guild ID. |
| User ID | Member to change. **Select** or **Map** the user's numeric id. |
| User Role | Role to remove. **Select** when server roles appear in the list. |

</Accordion>
<Accordion title="Channels: Find Channel">

Returns one channel or all channels.

| Field | Description |
| --- | --- |
| Guild ID | Server to search. **Select** or **Map** the guild ID. |
| Channel ID | Optional. Leave empty to return all channels, or set one channel via **Select** / **Map**. |
| Channel Name | Optional text filter by channel name. |

</Accordion>
<Accordion title="Channels: Rename Channel">

**Renames a channel** on the server.

| Field | Description |
| --- | --- |
| Guild ID | Server that owns the channel. **Select** or **Map**. |
| Channel ID | Channel to rename. **Select** from loaded channels or **Map** the numeric **channel id**. |
| New Channel Name | New name: type it in **Select** mode or **Map** from scenario data. |

</Accordion>
<Accordion title="Members: Find User">

Returns one member or all members.

| Field | Description |
| --- | --- |
| Guild ID | Server to search on. **Select** or **Map** the guild ID. |
| User ID | Optional. Narrow to one user: **Select** or **Map** their numeric user id. |
| Username | Optional. Filter by Discord username text. |

</Accordion>
<Accordion title="Stickers: List Stickers">

**Lists custom stickers** available on the server.

| Field | Description |
| --- | --- |
| Guild ID | Server whose stickers to list. **Select** or **Map** the guild ID. |

</Accordion>
<Accordion title="Messages: Send Direct Message">

**Sends a DM** to a user (opens the bot's DM channel).

| Field | Description |
| --- | --- |
| Guild ID | Server where the recipient is a member (the bot must be there too). **Select** or **Map** the guild ID. |
| User ID | Who receives the DM. **Select** if members load in the list; otherwise switch to **Map** and pass a user ID from a previous node or copy it in Discord (Developer Mode → right-click user → **Copy User ID**). |
| Message Content | Message text. Type it directly or **Map** from another node (for example a webhook or form field). |

</Accordion>
<Accordion title="Messages: Send Message">

**Posts a message** to a text channel on the server.

| Field | Description |
| --- | --- |
| Guild ID | Server. **Select** or **Map** the guild ID first so channel lists can load. |
| Channel ID | Target channel. **Select** from the dropdown when it fills, or **Map** the numeric **channel id**. |
| Message Content | Optional. Static text or **Map** dynamic content from the scenario. |

</Accordion>
<Accordion title="Messages: Send Message with Attachment">

**Sends a channel message with a file** from a prior node or URL.

| Field | Description |
| --- | --- |
| Guild ID | Server. **Select** or **Map** the guild ID. |
| Channel ID | Target channel. **Select** or **Map** like **Send Message**. |
| Message Content | Optional caption. Text or **Map**. |
| Attachment Name | File name shown in Discord. Often **Map** from prior output, e.g. `1.body.files.[0].filename`. |
| Attachment Path or URL | File body or link. **Map** e.g. `1.body.files.[0].content` or a reachable URL. |

</Accordion>
<Accordion title="Message components: Buttons JSON">

Builds a JSON block with 1 to 5 buttons for **Send Message**.

| Field | Description |
| --- | --- |
| Buttons Count | How many buttons (1 to 5). Extra **Button N:** fields follow this count. |
| Button N: Style | Visual style: Primary, Secondary, Link, and other Discord button types. |
| Button N: Label | Text on the button. |
| Button N: Custom ID | Id your scenario can read on click. Required for every style except **Link**. |
| Button N: URL | Required only for **Link** buttons (opens this URL). |
| Button N: Disabled | Optional. Turn off the button without removing it. |

Field count follows **Buttons Count**.

</Accordion>
</Accordions>

## Triggers

<Accordions type="multiple">
<Accordion title="New Member In Guild">

Fires when someone joins the server.

| Field | Description |
| --- | --- |
| Guild ID | Which server to watch. **Select** or **Map** the guild ID (bot must be on that server). |

</Accordion>
<Accordion title="New Message In Channel">

**Fires when a new message** is posted in the watched channel.

| Field | Description |
| --- | --- |
| Guild ID | Server. Set this first (**Select** / **Map**) so the channel list can load. |
| Channel ID | Channel to watch. **Select** when available, or **Map** the numeric **channel id**. |

</Accordion>
<Accordion title="New Message In Thread">

**Fires when a new message** appears in the watched thread.

| Field | Description |
| --- | --- |
| Guild ID | Server. **Select** or **Map** the guild ID. |
| Channel ID | Parent text channel of the thread. **Select** or **Map**. |
| Thread ID | Thread to watch. **Select** if listed, or **Map** the numeric **thread id**. |

</Accordion>
<Accordion title="Message components: Selectors JSON">

**Dropdown (select menu) JSON** for the **Send Message** node.

| Field | Description |
| --- | --- |
| Custom ID | Id your app receives when the user picks an option. |
| Placeholder | Optional hint text in the closed dropdown. |
| Minimum Values | Optional minimum number of options the user must pick. |
| Maximum Values | Optional maximum selections (up to Discord limits). |
| Options Count | How many menu options (1 to 25); matching option fields follow. |

</Accordion>
</Accordions>
