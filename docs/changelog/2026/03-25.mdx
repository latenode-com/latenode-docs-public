**Fast webhook response**

For [**Trigger on Webhook**](/integrations/core-nodes/trigger-on-webhook), you can enable fast response mode: the query parameter `__ln.fast=1` returns a reply immediately without waiting for the scenario to finish.

- Configure response **body**, **status**, and **headers** via GET parameters: `__ln.resp.body`, `__ln.resp.status`, `__ln.resp.header.*`
- Response time for asynchronous scenarios is reduced to roughly **~100 ms**
